# A111-APIS-0001: A1111 WebUI HTTP API 参考

| 属性 | 值 |
|------|-----|
| 分类 | A1111 WebUI API Reference |
| 版本 | v2 (Stable Diffusion WebUI by AUTOMATIC1111) |
| 来源 | `modules/api/api.py` + `modules/api/models.py` (master) |
| 框架 | FastAPI + uvicorn + Gradio queue |
| 协议 | HTTP REST + JSON |

## 概述

A1111 WebUI 内置 FastAPI 驱动的 HTTP API，所有端点位于 `/sdapi/v1/` 下。
生成端点为**同步阻塞式** — 请求挂起直到整个生成完成后返回结果。

## 完整端点列表

### 生成类 (Generation)

```json
  "do_not_save_grid": false,
  "eta": 0,
  "denoising_strength": 0,

  "override_settings_restore_afterwards": true,
  "refiner_checkpoint": "",
  "refiner_switch_at": 0,
  "disable_extra_networks": false,
  "firstpass_image": null,
  "enable_hr": false,
  "firstphase_width": 0,
  "firstphase_height": 0,
  "hr_scale": 2.0,
  "hr_upscaler": "",
  "hr_second_pass_steps": 0,
  "hr_resize_x": 0,
  "hr_resize_y": 0,
  "hr_sampler_name": "",
  "hr_scheduler": "",
  "hr_prompt": "",
  "hr_negative_prompt": "",
  "force_task_id": "",
  "sampler_index": "Euler",
  "script_name": null,
  "script_args": [],
  "send_images": true,
  "save_images": false,
  "alwayson_scripts": {},
  "infotext": null,
  "comments": {}
}
```

响应 (`TextToImageResponse`):

```json
{
  "images": ["base64..."],
  "parameters": { /* 回显请求参数 */ },
  "info": "{\"prompt\":\"...\",\"seed\":12345,...}"
}
```

- `images`: base64 编码图片列表 (不含 `data:` 前缀)，编码格式由 `samples_format` 决定 (png/jpeg/webp)
- PNG 内嵌元数据 text chunk；JPEG/WEBP 将参数写入 EXIF UserComment
- `infotext`: 若提供，会解析并填充缺失字段，支持覆盖 settings 和 script 参数
- `script_name` + `script_args`: 选择脚本及其参数
- `alwayson_scripts`: key 为脚本名，value 含可选 `args` 列表

#### `POST /sdapi/v1/img2img` — 图生图

请求体 (`StableDiffusionImg2ImgProcessingAPI`) 在 txt2img 基础上增加:

```json
{
  "init_images": ["base64..."],
  "mask": null,
  "mask_blur": 4,
  "mask_blur_x": 4,
  "mask_blur_y": 4,
  "mask_round": true,
  "mask_alpha": 0,
  "inpainting_fill": 0,
  "inpaint_full_res": true,
  "inpaint_full_res_padding": 0,
  "inpainting_mask_invert": 0,
  "initial_noise_multiplier": 1.0,
  "latent_mask": null,
  "include_init_images": false
}
```

响应 (`ImageToImageResponse`) 同 txt2img 格式，额外含 `init_images` 字段 (若 `include_init_images=true`)。

- `init_images`: 输入图片 (列表)，支持三种格式:
  - 纯 base64 字符串
  - data URI (`data:image/png;base64,...`)
  - HTTP(S) URL (需启用 `--api-allow-request-downloads`)

### 控制类 (Control)

#### `GET /sdapi/v1/progress` — 进度查询

查询参数: `?skip_current_image=true` (跳过当前图片 base64 返回以提升性能)

响应 (`ProgressResponse`):

```json
{
  "progress": 0.5,
  "eta_relative": 10.0,
  "state": {
    "skipped": false,
    "interrupted": false,
    "job": "txt2img",
    "job_count": 1,
    "job_no": 0,
    "sampling_step": 10,
    "sampling_steps": 20
  },
  "current_image": "base64...",
  "textinfo": "Steps: 20, Sampler: Euler a, ...",
  "id_task": "task-abc123"
}
```

- `current_image`: 当前步骤部分结果 base64 (若 `skip_current_image=true` 则为 null)
- 无生成任务时返回 `progress: 0, eta_relative: 0, state: {...}` 且无 `current_image`

---

#### `POST /sdapi/v1/interrupt` — 中断当前生成

无请求体，返回 `{}`。中断当前生成任务 (设置 `shared.state.interrupted = True`)。

---

#### `POST /sdapi/v1/skip` — 跳过当前步骤

无请求体，返回 `{}`。跳过当前采样步骤 (设置 `shared.state.skipped = True`)。

## 认证 (Authentication)

若启动参数含 `--api-auth user:password`，所有 API 端点强制 HTTP Basic 认证。
支持多组凭据：`--api-auth user1:pass1 user2:pass2`。

## 中间件行为

### API Logging

若 `--api-log` 已设置，每个请求记录: 时间戳、状态码、方法、端点、客户端 IP、耗时。

### 异常处理

所有未捕获异常由中间件统一转换为 JSON 错误响应:

```json
{
  "error": "RuntimeError",
  "detail": "",
  "body": "",
  "errors": "CUDA out of memory. Tried to allocate ..."
}
```

- `error`: 异常类名 (如 `RuntimeError`, `ValueError`, `HTTPException`)
- `detail`: 异常详情字符串
- `body`: 异常体字符串
- `errors`: 异常的完整字符串表示 (`str(exception)`)
- HTTP 状态码: 400 (HTTPException) 或 500 (其他异常)

若 `WEBUI_RICH_EXCEPTIONS` 环境变量已设置，异常时额外输出 rich traceback 到日志。

## 并发与队列

A1111 使用 `queue_lock` (threading.Lock) 序列化生成和 extras 操作:

- 同一时刻仅一个 txt2img/img2img/extra-* 请求在执行
- 后续生成请求由 Gradio 队列排队等待 (不会立即返回错误)
- `GET /sdapi/v1/progress` 不受 queue_lock 限制，可在生成期间正常响应
- 非生成类端点 (查询/配置等) 也不受 queue_lock 限制

---

## 关键特征

| 特征 | 说明 |
|------|------|
| **同步阻塞** | 生成端点同步等待完成，整个请求周期可能数秒到数分钟 |
| **Base64 传输** | 输入/输出图片均为 JSON 中的 base64 字符串 (不含 `data:` 前缀) |
| **队列调度** | 并发生成请求自动排队，不立即拒绝 |
| **FastAPI 驱动** | 底层为 FastAPI + uvicorn，非 Gradio 代理 |
| **Gradio 集成** | API 通过 `shared` 模块与 Gradio 共享状态和锁 |
| **无流式输出** | 无 SSE/WebSocket，结果一次性返回 |
